  • Project Management Discussion:

A project is a joint activity requiring both a defined duration and target cost. Projects consist of various phases during which varying resources produce specified results1. Marketing and business management perform an analysis to assure that the project's deliverable is viable for the business. The success of the project execution that follows will be strongly tied to the accuracy of the requirements and design specifications that define both the required function and performance2  of the offer.

Many companies define their formal project process as specific phases3, each phase having defined deliverables for the subsequent phase. The formality of the phase breakdown allows management to track project progress and cost against the plan, as well as to assure that each required activity for project completion is identified and attended to.

It is essential for management to be able to track the project's progress. Schedule and cost can be accounted for using work breakdown structures4. The WBS can be itemized into a tool such as Microsoft Project® to produce a waterfall execution path. This is true regardless of the use of Agile or Waterfall5 project methodology. Even short cycle 'sprint' deliverables of Agile can have cost and schedule work breakdown elements.
